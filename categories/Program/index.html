如何高效学习编程

我认为观看国内(Bilibili)平台的视频来学习又长又臭，比较浪费时间(二三十小时)，所以对此来写一篇文章来讲讲如何高效学习编程

前端方面

你可以刷刷FreeCodeCamp这个网站, 来学习HTML, CSS, JavaScript, 和一些框架的知识，配合MDN 文档，这可以让你快速入门这些基本技术。

等学完这些就可以全面的攻克框架了, 对于框架及一些比较新的知识, 我建议去看看 YouTube 或者 Udemy 的优质课程, 如果你要快速入门, 可以看看 YouTube 上的一些类似crash course的课程，前端方面我推荐一位YouTuber The Net Ninja 。

有关于crash course课程的介绍，这边引用一段知乎那边的回答

优点如下：
1.精炼简短。每集10分钟左右，一门学科也就30-50集。不到10小时就可以粗略了解一门学科。
2.风趣幽默。语言贴近生活，还有很多有意思的、入时的梗。
3.结构优良。整个视频制作浑然一体，有老师面授，有图像、视频资料，有逻辑图和图表，整个视频呈现出一种高度集成统一的状态。

不足如下：
1.深度有限。想要把一门学科学深学透，光靠看几个小时视频远远不够。
2.广度有限。因为资源获得问题，只能在B站看搬运视频，目测涉及学科和领域还比较有限。（希望up主们多多搬运）

当然，“不足”并非该系列视频的不足，单就视频本身来说是非常完美的。所谓不足，是说我们使用这套视频学习知识，需要认识到它的局限性而已。

简单的来讲就是快速的速成课，能让你快速的入门一门技术但是不精通，入门之后再看看官方文档，看看 Github 社区项目就可以进一步学习了。

后端方面

因为我目前的方向是前端，所以后端方面的也是刚学习，我目前在学习 Rust , 由于国内中文的教程很少，所以只能看看社区文档和配合一些国外的视频学习，这里推荐一个学习编程的社区 LearnKu 。

如果不清楚学习路线的话可以参考一下 roadmap.sh

以上是我自己的一些学习技巧，仅供参考。
